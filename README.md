# BulkApply

A self-hosted job application automation platform. Search for jobs, manage your pipeline, and send personalized bulk emails — all from one dashboard.

## Tech Stack

| Layer     | Technology                           |
|-----------|--------------------------------------|
| Backend   | PHP 8.2, Laravel 12                  |
| Frontend  | React 19, Inertia.js, Tailwind CSS 4 |
| Build     | Vite 7                               |
| Auth      | Laravel Sanctum (session + API tokens) |
| PDF       | smalot/pdfparser                     |
| Queue     | Laravel Queue (database driver)      |

---

## Getting Started

```bash
# Install and set up everything
composer run setup

# Start all dev processes (server, queue worker, logs, Vite)
composer run dev
```

Configure your `.env` before running:

```env
# Required for job search
ADZUNA_APP_ID=your_id
ADZUNA_APP_KEY=your_key
ADZUNA_COUNTRY=in          # in | us | gb | au | ...

APP_KEY=                   # generated by setup
DB_CONNECTION=sqlite       # default
```

## Features

### Authentication
- Register and login with email/password.
- Each user has an isolated profile — credentials and documents never cross accounts.

---

### Dashboard
- Summary cards: total, pending, queued, sent, failed application counts.
- Sent rate percentage.
- 30-day daily activity chart (added vs sent).
- This week vs last week comparison.
- Top 5 companies by application count.
- Recent activity feed (last 10 sent/failed).
- Email open rate tracking stats.
- Pipeline status breakdown (Applied, Replied, Interview, Rejected, Offer).

### Profile & Settings
Located at `/profile`.

**Personal info:** Full name, email, phone, location, preferred role.

**Document uploads:** Resume and cover letter (PDF/DOC/DOCX, up to 10 MB each).

**Resume auto-parse:** Upload a resume and the system extracts your name, email, phone, and skills automatically (supports PDF, DOCX, DOC).

**Email sender setup:** Connect your own Gmail account using an App Password. Credentials are encrypted at rest with `APP_KEY`. The password is write-only — never sent back to the browser.

**Email template (default):** Subject and body with placeholders (see below).

**Send scheduling:** Set a start hour, end hour, and optionally weekdays-only. Emails queued outside this window are held and retried every 5 minutes.

**Rate limiting:** Max emails per hour cap per account.

**Follow-up scheduling:** Set a number of days after sending; a follow-up email is dispatched automatically.

**Webhook:** A URL that receives a POST with event data (`application_sent` / `application_failed`) after each send.

**Preferred job sites:** Mark sites you prefer for the job search page.

**Skills:** A free-text skill list used for skill-match highlighting in job search results.

#### Email Placeholders

| Placeholder        | Replaced with                        |
|--------------------|--------------------------------------|
| `{job_title}`      | Job title (or "the role")            |
| `{company}`        | Company name                         |
| `{recruiter_name}` | Recruiter name (or "Hiring Manager") |
| `{location}`       | Job location                         |
| `{job_url}`        | Job listing URL                      |
| `{your_name}`      | Your full name                       |
| `{your_location}`  | Your location                        |
| `{your_email}`     | Your email                           |
| `{your_phone}`     | Your phone                           |

- Upload multiple resumes (PDF/DOC/DOCX).

- Delete individual resumes.

### Resume ATS Checker
Located at `/resume-check`.

Scores your uploaded resume 0–100 for ATS (Applicant Tracking System) friendliness. Pure PHP — no external API.

**Checks performed (with weighted scores):**

| Category      | Check                         | Weight |
|---------------|-------------------------------|--------|
| File & Format | File format (PDF/DOCX)        | 10     |
| File & Format | Machine-readable text         | 15     |
| File & Format | File name not generic         | 4      |
| File & Format | No tables (DOCX only)         | 7      |
| File & Format | No embedded images (DOCX)     | 5      |
| Contact Info  | Email address present         | 10     |
| Contact Info  | Phone number present          | 6      |
| Contact Info  | LinkedIn / GitHub link        | 4      |
| Structure     | Standard section headings     | 15     |
| Structure     | Professional summary          | 4      |
| Structure     | Resume length (300–1000 words)| 6      |
| Structure     | Employment dates present      | 5      |
| Content       | Action verbs                  | 7      |
| Content       | Quantified achievements       | 7      |
| Content       | Bullet points                 | 4      |
| Keywords      | Target role keyword match     | 10     |

**Grades:** Poor (< 50) / Needs work (50–69) / Good (70–84) / Excellent (85+)

Each check returns a pass / warn / fail status with an actionable improvement tip.

### Email Templates
Located at `/templates`.

- Create, edit, and delete named email templates.
- Each template has a name, subject, and body — all support the same `{placeholders}` as the profile template.
- Mark one template as the **default** — it is used for all bulk sends unless overridden.
- When triggering a bulk send, choose a specific template from a dropdown (overrides the default).
- Preview any template rendered against a real job record before sending.

---

### Job Applications
Located at `/jobs`.

#### Adding Jobs

**Manual entry:** Add a single job with company, job title, recruiter name, recruiter email, job URL, location, and notes.

**CSV import:** Upload a CSV file. Column names are aliased (e.g. `email`, `contact_email`, `recruiter_mail` all map to `recruiter_email`). Duplicate detection skips rows where the same company + email already exists. Invalid rows (missing company or email) are counted and reported.

**CSV template:** Download a ready-to-fill template from `/jobs/template`.

**From job search:** Import selected search results directly from the Find Jobs page (see below).

**Via Chrome extension:** Save jobs from any job board using the browser extension (see Extension section).

#### Job List

- Search across company, job title, recruiter name, email, and location.
- Filter by send status (pending / queued / sent / failed) or pipeline stage.
- Sort by any column (company, job title, status, pipeline status, created date, sent date).
- Active jobs (pending/queued/failed) always surface above sent ones.
- Count badges for total, pending, sent, and failed.

#### Sending Applications

**Bulk send:** Queues all pending/failed jobs as a Laravel batch. Progress is tracked live (total / processed / failed / pending). The active batch can be cancelled — already-dispatched jobs finish, remaining queued ones revert to pending.

**Send one:** Queue a single job independently.

**Template selection:** Pick any email template at send time; overrides the default.

**Email preview:** Preview the rendered subject, body, and recipient for any job before sending.

#### Per-Job Actions

- **Retry:** Re-queue a failed job via "Send".
- **Delete:** Remove a single job.
- **Pipeline update:** Move a job through stages: Applied → Replied → Interview → Rejected / Offer.
- **Error display:** Failed jobs show a truncated error message inline; full error visible on hover/expand.

#### Export

Download all applications as a CSV (`/jobs/export`) including tracking timestamps, pipeline status, follow-up count, and all job metadata.

#### Clearing

Clear all job applications at once (destructive, requires confirmation in the UI).

### Find Jobs (Job Search)
Located at `/search`.

Search for live job listings and import them directly into your pipeline.

**Search modes:**

1. **General search** — searches across the whole web via the Adzuna API. Supports role, location, sort order (relevance / date / salary), and full-time filter.
2. **Site-specific search** — enter a site name or URL:
   - Recognised tech parks (Infopark, Technopark, Cyberpark) are scraped directly from their job boards.
   - Big platforms (LinkedIn, Indeed, Naukri, etc.) that block scraping fall back to the aggregated API with a note.
   - Any other site/URL is used as an extra keyword in the aggregated search.

**Skill matching:** Each result is scanned for known skills (from a dictionary of 100+ technologies). Skills that match your profile skill list are highlighted; unmatched skills shown separately.

**Contact enrichment:** For jobs without a recruiter email, the system probes the company's own website (guessed from the company name) to find a real published email (careers@, hr@, jobs@, etc.). Only same-domain emails are used; free-mail domains and placeholder addresses are filtered out.

**Auto-apply:** Select results and click "Auto-apply". Jobs with a found email are immediately queued for email sending; link-only jobs are saved as pending with a note to apply manually.

**Resume selection:** Choose which of your uploaded resumes to attach when auto-applying from search.

**Search preferences:** Role and location are saved to your profile for next time.

#### Supported Job Sources

| Source      | Method      | Location                  |
|-------------|-------------|---------------------------|
| Adzuna API  | REST API    | Global (country-scoped)   |
| Infopark    | HTML scrape | Kochi, Kerala             |
| Technopark  | HTML scrape | Trivandrum, Kerala        |
| Cyberpark   | HTML scrape | Kozhikode, Kerala         |
| Any site    | Keyword fallback | Aggregated via Adzuna |

### Email Tracking
Fully automatic — no configuration needed.

Each outgoing email embeds:
- A **1×1 tracking pixel** (`/track/pixel/{id}`) — records `opened_at` when the recruiter opens the email.
- A **tracked link** (`/track/click/{id}`) — records `clicked_at` and redirects to the job URL when clicked.

Tracking endpoints are public (no auth) so they work from any email client.

Open and click timestamps appear in the job list and on the dashboard.

---

### Follow-up Emails
- Set `followup_days` in Settings (0 = disabled).
- After a successful send, `followup_at` is set to `now + followup_days`.
- Run the scheduler command to dispatch follow-ups:
  ```bash
  php artisan app:dispatch-followups
  ```
- Only jobs in "applied" pipeline status with a past `followup_at` get a follow-up.
- Each job tracks a `followup_count`.

---

### Webhook Notifications
Set a webhook URL in Settings. After each send attempt, BulkApply POSTs:

```json
{
  "event": "application_sent | application_failed",
  "company": "...",
  "job_title": "...",
  "email": "...",
  "status": "sent | failed",
  "sent_at": "ISO 8601 or null",
  "error": "null or error message"
}
```

Webhook failures are silently ignored so they never cause an application send to fail.

### Chrome Extension
Located at `/extension`.

A companion browser extension lets you save jobs from any job board directly to your BulkApply dashboard without leaving the page.

**Supported job sources:** LinkedIn, Indeed, Naukri, Glassdoor, Greenhouse, Lever. Each site has a dedicated adapter in `extension/content.js` that scrapes company, job title, location, and description using selectors specific to that site's DOM.

**API endpoints (token-based via Laravel Sanctum):**

| Method | Endpoint                                          | Description                          |
|--------|----------------------------------------------------|---------------------------------------|
| POST   | `/api/extension/login`                            | Authenticate, returns token           |
| POST   | `/api/extension/jobs`                              | Save a job to dashboard               |
| GET    | `/api/extension/profile`                           | Fetch name/email/phone/resume for autofill |
| POST   | `/api/extension/jobs/{job}/auto-apply-status`      | Report Easy Apply outcome for a job   |

The extension saves company, job title, job URL, location, description, and source. Duplicate detection matches on company + job title, or on `job_url`, per user. Saved jobs land in the job list with `apply_type = link`, unless scraped from a LinkedIn Easy Apply listing, in which case `apply_type = easy_apply` (excluded from bulk email sending — see below).

**LinkedIn Easy Apply assist:** On a LinkedIn job with an Easy Apply button, the popup shows an "Auto-fill Easy Apply" action (`extension/easyapply.js`, relayed through `extension/background.js`). It opens the application modal and fills what it can confidently match from your profile — name, email, phone, and an existing resume if offered — using label/aria-based field matching, then **stops before the final Submit**. LinkedIn's ToS prohibits automated interaction, and custom per-job screening questions aren't safe to guess, so the assist leaves review and submission to the user and flags any fields it couldn't map. It cannot attach a new resume file (browsers block scripted file-input values). The outcome (`filled` / `submitted` / `failed`) is recorded on the job via `auto_apply_status`/`auto_apply_error`/`applied_at`.

The prebuilt `public/downloads/bulkapply-extension.zip` has no build step — it must be manually rebuilt (`zip -j`) from the `extension/` folder after any change to those files.

---

### Queue Management

**Bulk send batching:** Uses Laravel's `Bus::batch()`. Each batch is tracked by ID on the profile. The UI polls for progress and shows live counts.

**Rate limiting:** Configured per-account via `RateLimited('mail-sending')` middleware on the `SendJobApplication` job. Jobs that hit the limit are released (not failed) and retried for up to 3 days.

**Scheduling window:** Jobs released outside the configured send window are retried every 5 minutes until inside the window.

**Stale queue sweep:** Resets applications stuck in `queued` status for more than 30 minutes back to `pending` (handles crashed workers or abandoned batches):
```bash
php artisan app:sweep-stale-queued
```

**Recommended cron (add to your server's crontab):**
```
* * * * * cd /path/to/bulkappply && php artisan schedule:run >> /dev/null 2>&1
```

```
pending  -->  queued  -->  sent
                |
                +--->  failed  -->  (retry) --> queued --> sent
```

```
applied --> replied --> interview --> offer
                              \
                               --> rejected
```
